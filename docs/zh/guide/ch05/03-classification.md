# 5.3 分类和检测

## 这一节解决什么问题
这一节解决的是：**当我们不需要逐像素勾轮廓时，如何回答“有没有病”“是什么病”“大概在什么位置”。**

## 为什么前面几章还不够
前两节已经处理了“输入怎么准备”和“边界怎么分出来”，但很多真实场景并不直接需要精细 mask：

- 胸片初筛只想先判断是否异常；
- 急诊场景更关心高召回的阳性提示；
- 大规模体检先要快速分流，而不是先做精细勾画。

所以，分类与检测承担的是**先判断、再定位、再交给后续流程细化**的角色。它们通常是临床 AI 工作流的第一道门。

---

## 为什么需要
分类和检测最擅长解决三类问题：

| 任务 | 输出形式 | 典型问题 |
| --- | --- | --- |
| 分类 | 标签或概率 | 正常 / 异常、疾病类型、多标签共病 |
| 检测 | 边界框 + 类别 | 病灶在哪、是否有多个候选病灶 |
| 分类 + 可解释性 | 概率 + 热力图 | 模型为什么做出这个判断 |

在医学场景里，分类与检测尤其强调三件事：

1. **高召回**：宁可多报，也不能轻易漏掉严重病灶。
2. **类别不平衡处理**：阳性常常远少于阴性。
3. **可解释性**：模型输出必须能被医生追问和核查。

---

## 核心直觉
分类模型的核心不是“记住器官长什么样”，而是**在全局图像里找到与诊断相关的模式**：

- 胸片分类更偏向全局结构与局部异常的共同判断；
- CT/MRI 分类可能依赖多层切片或多序列信息；
- 检测任务会在分类能力之外，再学习“病灶在哪儿”。

一个很实用的理解方式是：

- **分类**回答“有没有/是什么”；
- **检测**回答“在哪里”；
- **分割**回答“边界到哪里”。

## 典型场景

### 场景 1：胸部 X 光二分类或多标签分类
- **目标**：正常/异常、肺炎、积液、结节等标签预测。
- **难点**：阳性比例低，且很多异常只占很小区域。
- **本地源码**：`src/ch05/medical_image_classification/main.py`。

### 场景 2：病灶检测作为分诊入口
- **目标**：给出候选病灶框，供医生或后续分割模型复核。
- **适合**：肺结节、乳腺钙化、骨折可疑点等。
- **本节重点**：先建立分类直觉，再把“分类头”扩展成“框回归头”。

### 场景 3：模型解释与错误分析
- **目标**：不仅看准确率，还要知道模型到底看到了哪里。
- **建议输出**：预测概率、混淆矩阵、ROC/AUC、热力图或注意力图。
- **本地结果文件**：`src/ch05/medical_image_classification/output/medical_classification_report.json`。

## 最小流程
一个可落地的医学分类最小流程通常如下：

1. **准备样本与标签**：单标签、多标签或分层标签。
2. **统一输入形状与强度范围**：保持训练/验证/测试一致。
3. **构建 backbone + 分类头**：常见如 ResNet、DenseNet、ViT。
4. **处理类别不平衡**：重采样、加权损失、阈值重设。
5. **输出指标与解释结果**：不要只报告 accuracy。

常见输入输出约定：

- **输入**：`[B, C, H, W]` 图像张量。
- **输出**：`[B, K]` 的类别 logits 或概率。
- **附加输出**：注意力图、置信度、不确定性估计。

## 片段代码
正文只保留帮助理解的关键片段；完整网络、训练循环和结果可视化请看本地脚本。

### 1. 最小分类头
```python
import torch.nn as nn


def classification_head(in_features, num_classes):
    return nn.Sequential(
        nn.Linear(in_features, 256),
        nn.ReLU(inplace=True),
        nn.Dropout(0.5),
        nn.Linear(256, num_classes),
    )
```

- **输入**：骨干网络提取到的全局特征向量。
- **输出**：每个类别的 logits。
- **完整实现**：`src/ch05/medical_image_classification/main.py`。

### 2. 类别不平衡时的加权交叉熵
```python
import torch
import torch.nn.functional as F


def weighted_ce(logits, targets, class_weights):
    return F.cross_entropy(logits, targets, weight=torch.tensor(class_weights))
```

- **输入**：模型输出、真实标签、类别权重。
- **输出**：更关注少数类的损失值。
- **适用**：正常样本远多于异常样本的场景。

### 3. 从 logits 到可读概率
```python
import torch


def to_probabilities(logits):
    return torch.softmax(logits, dim=1)
```

- **输入**：未归一化的模型输出。
- **输出**：每类概率，便于阈值分析与可解释化展示。
- **本地报告**：`src/ch05/medical_image_classification/output/medical_classification_report.json` 中给出了示例指标。

## 延伸实践
1. **先阅读并运行** `src/ch05/medical_image_classification/main.py`，关注数据生成、网络结构和评估输出的对应关系。
2. **不要只看 accuracy**：同时比较 precision、recall、F1、AUC，以及不同阈值下的表现。
3. **做一次错例分析**：把假阳性和假阴性单独可视化，往往比单个总分更有启发。
4. **从分类过渡到检测**：把“全图标签”改成“候选框 + 类别”，你会更清楚检测比分类多学了什么。
5. **增加解释模块**：给模型接一个 Grad-CAM 或注意力热图，检查它是否真的关注病灶区域。

理解了这一节之后，你会更容易判断：某个任务到底应该先做分类、直接做检测，还是需要再走到分割那一步。
